Deep Focus · Build Guide

Get Your APK
in 5 Minutes

No PC needed. GitHub builds the APK for free on their servers. Follow these steps on your phone or any device.

1

Create a free GitHub account

Go to github.com and sign up. It's completely free. If you already have an account, just sign in.

https://github.com/signup
2

Create a new repository

After signing in, click the + icon (top right) → New repository.

Settings to use:
• Repository name: deep-focus
• Visibility: Public (required for free Actions minutes)
• ✅ Check "Add a README file"
• Then click Create repository
3

Upload all the project files

In your new repository, click Add file → Upload files. Then drag and drop all the files and folders from the ZIP I gave you.

Important: GitHub's upload page lets you drag entire folders. Drag the whole DeepFocusRepo folder contents — not the folder itself, but everything inside it.

You need to upload these key items:

app/
.github/
gradle/
build.gradle.kts
settings.gradle.kts
gradlew
gradlew.bat
.gitignore
⚠️ The .github/workflows/build.yml file is critical — it's what triggers the automatic build. Make sure it uploads correctly.
4

Commit the files

After uploading, scroll down on the upload page. You'll see a Commit changes section. Leave the default message and click the green Commit changes button.

✅ The moment you commit, GitHub automatically starts building your APK. This usually takes 3–6 minutes.
5

Watch the build run

Click the Actions tab at the top of your repository. You'll see a workflow called "Build Debug APK" running with a yellow spinning circle.

Status meanings:
🟡 Yellow spinning = Building (wait ~5 min)
✅ Green checkmark = Build succeeded! APK ready.
❌ Red X = Build failed (contact Aditya with the error log)
6

Download your APK

Once the build shows a green checkmark:

1. Click on the workflow run name
2. Scroll down to the Artifacts section
3. Click DeepFocus-debug-apk to download a ZIP
4. Unzip it — inside is your app-debug.apk file

🎉 That's your APK! Transfer it to your Android phone via WhatsApp, Telegram, Google Drive, or USB and install it.
7

Install on your phone

Android blocks unknown sources by default. To install:

1. Open the APK file on your phone
2. Tap Settings when prompted
3. Enable Install from this source
4. Go back and tap Install

⚠️ After installing, open the app and grant Do Not Disturb permission when asked. Without it, DND silencing won't work (the focus lock still works).

Common Questions

The build failed with a red X. What do I do?
Click the red X → click the failing step to expand the log → scroll to the bottom where the error is in red text. Take a screenshot and share it.
I don't see the Artifacts section after the build.
Make sure you click on the specific workflow run (the row with the commit message), not just the Actions tab overview. Artifacts appear at the very bottom of the run summary page.
Can I upload files from my phone directly to GitHub?
Yes! GitHub's mobile website works fine. Use your phone browser (Chrome/Firefox), go to github.com, and follow the same steps. The upload page accepts folders on mobile too.
How long does the APK artifact stay available?
30 days. After that you'd need to re-run the workflow (one click in the Actions tab → "Run workflow" button). You can also re-trigger it anytime by making any small commit.
Is this APK safe to install?
Completely. The source code is all yours — you can read every line of it. The build happens on GitHub's official servers. No external dependencies except standard Google/Android libraries.
